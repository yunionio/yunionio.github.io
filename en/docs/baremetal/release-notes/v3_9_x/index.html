v3.9.5​

date: 2022-12-06

Feature Enhancements (Enhancements)​

  • 【Host】Live migration optimization: shorten VPC host live migration network interruption time; compatible with old version host live migration; live migration fast convergence; batch live migration deadlock;
  • 【Host】Automatically ignore HUB type USB devices when detecting host USB devices
  • 【Host】Private cloud host monitoring data sent to metadata (169.254.169.254), not dependent on whether VMs can access influxdb
  • 【Host】Automatically identify virtual disks allocated on SSD type storage as SSD type
  • 【Host】Improved VM disk storage replacement UI interaction
  • 【Host】When synchronizing VMware VMs, if VMs with duplicate UUIDs are found, no longer terminate synchronization, will automatically ignore VMs with duplicate UUIDs
  • 【Host】When synchronizing VMware VMs, allow VM IP subnets to be unmanaged. If IP subnets are unmanaged, IP subnets will be synchronized to metadata, displayed on frontend and prompt users to manage IP subnets
  • 【Bare Metal】Allow specifying external tftp server and pxe boot files
  • 【CloudID】Improved HCSO SAML provider
  • 【Ceph】Prevent service hang issue when calling ceph commands
  • 【Ceph】Improved Ceph capacity calculation rules, use Used+MaxAvail as total capacity
  • 【API Gateway】Optimized floating point output format for excel export
  • 【Monitoring】Only check aliyun cloud account balance
  • 【Container】Deployed Kubernetes clusters support automatic certificate updates
  • 【Container】Deploy monitoring components for specified Kubernetes clusters

Bug Fixes (Fixes)​

  • 【Host】AWS VM CPU architecture synchronization uses instance's Architecture field
  • 【Host】Automatically generate ssh host keys for hosts, solving issue where some Ubuntu/Debian images generated VMs unable to ssh after startup
  • 【Host】VPC host inbound security group rules should default deny all traffic
  • 【Host】Fixed CPU architecture and UEFI format detection errors
  • 【Host】VMWare hosts converted to KVM hosts cannot be converted again
  • 【API Gateway】Error when uploading images via apigateway type endpoint
  • 【Scheduling】Should not check target host storage capacity limits when migrating hosts using shared storage
  • 【Bare Metal】Unused NICs missing index or nic_type, causing host conversion back to bare metal failure
  • 【i18n】Fixed error of repeatedly adding and deleting i18n information
  • 【Multi-Cloud Management】Prevent Cloudpods from nested managing itself
  • 【Image】arm images displayed as x86 CPU architecture after upload

New Features (New Features)​

  • 【Host】Allow updating host hostname
  • 【Host】Support creating disks from snapshots
  • 【Bare Metal】Allow managing bare metal without BMC
  • 【Container】Support deploying Kubernetes 1.20 cluster based on Galaxy Kylin V10SP2
  • 【Container】Automatically add nginx ingress controller for deployed container clusters, and support sticky session and other features
  • 【Monitoring】Automatically collect each service's performance metric parameters for subsequent service health monitoring
  • 【Deployment】ocboot supports automatic clickhouse setup
  • 【Frontend】Global search supports searching external IDs
  • 【Frontend】Add "About" page
